## 输入格式

- 账单周期从前 10 行含 `账单日` 的行取 `YYYY-MM-DD`。
- **交易行正则**：`^(\d{8})\s+(\d{8})\s+(\d{4})\s+(.+?)\s*CNY\s*([-\d.]+)\s+CNY\s*([-\d.]+)$`
- 格式：`交易日(YYYYMMDD) 记账日(YYYYMMDD) 卡号后4位 描述 CNY金额 CNY金额`
- **日期取记账日**（第二个），不是交易日。
- 示例：`20251214 20260114 2359 美团 CNY -87.50 CNY -87.50`

## 金额正负规则

**与储蓄卡相反**：
- **正金额 = 支出**（消费，增加应还款）
- **负金额 = 收入**（返现/优惠，减少应还款）

## 交易类型判定（`_process_refunds`）

| 条件 | 处理 |
|------|------|
| description 含 `还款` **且不含** `还款日`，负金额 | 存入 `repayments` → 产出 `__REPAYMENT__` 标记 |
| description 含 `还款` 但金额为正 | 直接跳过 |
| 负金额（非还款） | 收入「其他收入/抢红包」 |
| 正金额 + 退款关键词（`退款`/`退货`/`撤销`） | `refunds` 待对冲 |
| 正金额（其他） | 正常消费支出 |

## 还款标记机制（关键）

中信**独有**：负金额还款记录被保留为 `category="__REPAYMENT__"`、`subcategory="还款"`、`transaction_type="收入"`。

这些记录**不在最终输出中出现**，仅供 `merge.py` 用金额+日期（±3天）匹配储蓄卡的还款支出，匹配成功后两端删除并生成一条「转账-还款」记录。

未匹配的 `__REPAYMENT__` 在 merge.py 最终输出时也会被清理。

**与其他信用卡对比**：浦发/招商/建行信用卡的还款是直接跳过，**只有中信保留标记**。

## 商户提取与退款对冲

- 移除前缀：`支付宝－`、`支付宝-`、`财付通－`、`财付通-`、`微信支付－`、`Huawei Pay-`
- 正则去掉 `退款|退货|撤销`
- 匹配条件：**同商户 + 金额差 < 0.01** → 双方删除
- 未匹配的退款转「其他收入/退款」（描述追加 ` (未匹配退款)`）

## 支出分类规则（`_categorize`）

desc_lower 子串匹配：

| 描述含 | 分类 | 子分类 |
|--------|------|--------|
| `拉扎斯` / `美团` / `饿了么` / `肯德基` / `麦当劳` / `kfc` / `瑞幸` / `咖啡` / `coffee` | 食品酒水 | 早午晚餐 |
| `拼多多` / `淘宝` / `天猫` / `京东` | 居家物业 | 日常用品 |
| `母婴` / `babycare` / `童心` | 居家物业 | 日常用品 |

未命中走 `match_category`。

## PDF 解析特殊处理

**描述跨行问题**：PDF 抽取时描述可能落在交易行的**上一行**。用 `pending_description` 缓存"看起来像描述的非交易行"，下一行匹配到交易行时拼到 description 前缀。当跳过非交易行时清空 pending_description，避免残留。

**跳过非交易行的关键词黑名单**：
`账单日、Statement、卡号、第、页、【、】、CNY交易、交易日、Trans Date、信用额度、可用额度、到期还款日、本期应还、最低还款、账单周期、Min.、Payment、Balance、Charge、Previous、Card Number、Description、Trx.Amt、Setl.Amt、本期账单`

**额外跳过模式**：
- 卡号余额行：`^\d{4}[-*\d]+\s+CNY\s+[\d.]+\s+[\d.]+`
- 含 ≥3 个金额的多 CNY 行
- `^CNY\s+[\d.]+` 行
- 纯数字行
- 卡号行：`\d{4}[-*]+\d`
