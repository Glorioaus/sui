## 输入格式

- `pd.read_excel(file_path, skiprows=16)` —— **跳过前 16 行头部说明**。
- 账单周期从文件名 `\((\d{8})-(\d{8})\)` 提取。
- 列名（中文表头）：`交易时间`、`交易类型`、`交易对方`、`商品`、`收/支`、`金额(元)`、`支付方式`、`当前状态`、`备注`。

## 金额与收支规则

- 金额解析：`re.sub(r'[^\d.]', '', str)` 去 `¥` 等，float 转换；为 0 跳过。
- **方向由 `收/支` 列决定**：`支出` → 支出；`收入` → 收入；`/` → 跳过（通常是内部转账）。

## 银行卡识别常量

- `BANK_KEYWORDS = ["银行", "信用卡"]` —— 支付方式含任一即视为银行卡支付。
- `WALLET_METHODS = ["零钱", "零钱通"]` —— 微信钱包支付方式（实际代码中仅注释用）。

## 支付方式处理

| 支付方式 | 处理 |
|---------|------|
| 零钱 / 零钱通 | 保留，记入"微信"账户 |
| 银行卡支出 | **跳过**（避免与银行账单重复） |
| 银行卡转入零钱通 | 标记为转账（银行 → 微信） |

## 亲属卡标记机制（关键，跨账单关联）

触发条件：`交易类型` 含 `亲属卡` **且** `收/支="支出"`。

不论支付方式，都生成**标记交易**：
- `category="__FAMILY_CARD__"`
- `transaction_type="__MARKER__"`
- `subcategory=使用者名`（取 `交易对方`，为空或 `/` 时取 `"亲属"`）
- `account=银行名 or "__ANY_BANK__"`
- `merchant=使用者名`
- `description="亲属卡交易 {使用者}"`

**银行名提取**（`_extract_bank_name`）：
- 含 `储蓄卡` → `re.match(r'(.+?)储蓄卡', payment_method)` 取组1
  - 例：`农业银行储蓄卡(1970)` → `农业银行`
- 含 `信用卡` → `re.match(r'(.+?)银行信用卡', payment_method)` 取组1 + `信用卡`
  - 例：`中信银行信用卡(2359)` → `中信信用卡`

这些标记**不在最终输出中出现**，仅供 merge.py 的 `process_family_card` 用日期+金额匹配银行卡实际扣款，匹配成功后把那笔银行卡交易重分类为「其他杂项-{使用者}支出」。

## 描述拼接

`description = 交易对方（非"/"时前置）+ 交易类型 + 商品（非"/"时追加）`。`merchant = 交易对方`。

## 分类规则（`_apply_wechat_rules`）

text = `交易类型 + 交易对方 + 商品` 小写匹配：

**收入侧**：

| 描述含 | 分类 | 子分类 |
|--------|------|--------|
| `红包` | 其他收入 | 抢红包 |
| `退款` | 其他收入 | 退款 |
| `转账` | 其他收入 | 微信转账收入 |

**支出侧**：

| 描述含 | 分类 | 子分类 |
|--------|------|--------|
| `红包` | 人情往来 | 送礼请客 |
| `转账` | 人情往来 | 送礼请客 |
| `食堂` 或 `餐` | 食品酒水 | 早午晚餐 |
| `停车` | 行车交通 | 停车费 |
| `滴滴` / `出行` / `打车` | 行车交通 | 打车租车 |
| `美团` / `饿了么` / `外卖` | 食品酒水 | 早午晚餐 |

未命中走 `match_category`。
