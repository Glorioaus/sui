## 输入格式

- 账单周期从前 5 行 `(\d{4})年(\d{1,2})月` 取，默认年份 2025。
- **交易行正则**：`^(\d{1,2}/\d{1,2})\s+(\d{1,2}/\d{1,2})\s+(.+?)\s+([-\d.]+)\s+(\d{4})\s*(.*)?$`
- 格式：`MM/DD MM/DD 描述 金额 卡号后4 原始金额`
- 示例：`12/06 12/07 财付通-Manner Coffee 25.00 2116 25.00(CN)`
- **日期取记账日**（第二个）。

## 金额正负规则

- **正金额 = 消费支出**
- **负金额 = 退款 或 优惠/红包**

## 交易类型判定（`_process_refunds`）

| 条件 | 处理 |
|------|------|
| description 含 `还款` | **跳过**（不保留标记，与浦发一致，区别于中信） |
| 负金额 且 命中 `优惠` / `红包` / `抵扣` / `返现` / `奖励` | 收入 → 「其他收入/抢红包」 |
| 负金额（其他） | `refunds` 退款 |
| 正金额 | 正常消费支出 |

## 商户提取与退款对冲

- 移除前缀：`财付通-`、`支付宝-`、`微信支付-`、`云闪付-`
- **特殊处理「掌上生活优惠商户」**：用 `re.search(r'】(.+)$', description)` 提取 `】` 后的商户名。
  - 例：`掌上生活优惠商户-【月度新户专享】肯德基` → 商户名 `肯德基`
- 再正则去 `退款|退货|撤销`
- 匹配条件：**同商户 + 金额差 < 0.01** → 双方删除

## 支出分类规则（`_categorize`）

| 描述含 | 分类 | 子分类 | 备注 |
|--------|------|--------|------|
| `肯德基` / `麦当劳` / `kfc` / `manner` / `coffee` / `咖啡` / `饿了么` / `美团` / `叮咚` / `盒马` / `买菜` / `餐` / `食` | 食品酒水 | 早午晚餐 | |
| `停车` | 行车交通 | **停车费** | 子分类条件分支 |
| `加油` / `滴滴` / `高德` / `打车` / `出行` / `地铁` / `公交` | 行车交通 | **打车租车** | 含"停车"则用上一行子分类 |
| `淘宝` / `天猫` / `京东` / `拼多多` / `小红书` | 居家物业 | 日常用品 | |
| `会员` / `vip` / `订阅` | 休闲娱乐 | 会员 | |

未命中走 `match_category`。

## 跨年处理

若月份=1 且头部含 `year12月`（小写），年份+1。处理跨年账单（如 12 月账单跨到次年 1 月）。
