# 建设银行储蓄卡（CCB Debit）

建设银行储蓄卡明细有两种导出，**当前真实数据为 PDF**，优先用 PDF 解析器；CSV 解析器为遗留保留。

| 格式 | 解析器 | 文件名触发 | 状态 |
|------|--------|-----------|------|
| **PDF** | `src/parsers/ccb_debit_parser.py` → `CCBDebitParser` | `建行*.pdf`（非 `建行信用卡*.pdf`）| ✅ 主用 |
| CSV | `src/parsers/ccb_parser.py` → `CCBParser` | `建行*.csv` | ⚠️ 遗留 |

`main.py` 路由顺序：`建行信用卡.*\.pdf$`（信用卡）→ `建行(?!信用卡).*\.pdf$`（储蓄卡 PDF）→ `建行.*\.csv$`（储蓄卡 CSV）。

## PDF 格式（CCBDebitParser，主用）

- 用 `pdfplumber.extract_tables()` 提取 7 列表格（折行已合并进单元格）：
  `序号 | 摘要 | 交易日期 | 交易金额 | 账户余额 | 交易地点/附言 | 对方账号与户名`
- `account = 建行储蓄卡`；金额取 `abs()`；`is_income = amount >= 0`。
- 分类规则（保守，命中即用，否则走 `match_category`）：
  - 利息存入 → 职业收入-利息收入
  - 基金赎回 / 零钱通 / 余额宝 → 走 match_category（命中 理财-基金 / 理财-余额宝）
  - ATM存款 / 银联入账 → 收入默认（其他收入-意外来钱）
  - 消费 + 商户关键词：美团/饿了么/拉扎斯/咖啡/买菜… → 食品酒水-早午晚餐；淘宝/京东/拼多多 → 居家物业-日常用品；滴滴/高德 → 行车交通-打车租车
  - 跨行转出/转出/转账 + 礼金关键词（礼金/生日/出生/结婚…）→ 人情往来-送礼请客
  - 其余 → match_category

## CSV 格式（CCBParser，遗留）

- GBK 编码，`csv.reader`；列假设 `row[0]=日期、row[5]=金额、row[6]=描述`。
- ⚠️ 该列假设与现行建行导出不符（现行 PDF 为 7 列、日期在第 2 列），CSV 路径**未经真实数据验证**，仅作遗留保留。
- `account = 建行储蓄卡`；金额取 `abs()`；`is_income` 走对应映射。

## 注意

- 建行储蓄卡的还款支出现在能被 `merge.py` 的转账识别处理（`建行储蓄卡` 在 `DEBIT_ACCOUNTS` 中）。但「还**建设**银行信用卡」这类描述需 `建设银行信用` 关键词（已在 `TRANSFER_KEYWORDS` 补充），否则会落通用「信用卡」。
