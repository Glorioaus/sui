# 浦发信用卡（SPDB）

- **格式**：PDF
- **解析器**：`src/parsers/spdb_parser.py` → `SPDBParser`
- **账户名**：`浦发信用卡`
- **文件名触发**：`浦发*.pdf` 或 `*账单*.pdf`（通用兜底）

## 输入格式

- 账单周期从前 5 行 `(\d{6}).*账单` 取 6 位（YYYYMM）。
- **交易行正则**：`^(\d{8})\s+\d{8}\s+(.+?)\s+(\d{4})\s+[¥￥]?([-\d.]+)\s+([-\d.]+)\(CNY\)`
- 格式：`交易日 记账日 描述 卡号 ¥金额 金额(CNY)`

## 金额正负规则

- **正金额 = 支出**（消费）
- **负金额 = 退款或红包抵扣**

## 交易类型判定（`_process_refunds`）

| 条件 | 处理 |
|------|------|
| description 含 `还款` | **直接跳过**（不保留标记，区别于中信） |
| 负金额 且（`分润金` ∈ desc 或 `抵扣` ∈ desc） | 收入（红包抵扣）→ 「其他收入/抢红包」 |
| 正金额 且（`分润` ∈ desc 且 `退还` ∈ desc） | 当作支出（红包退还，抵消之前的收入） |
| 正金额（其他） | 正常消费支出 |
| 负金额（其他） | `refunds` 退款对冲 |

未匹配退款 → 「其他收入/退款」（追加 ` (未匹配退款)`）；红包 → 「其他收入/抢红包」。

## 商户提取与退款对冲

- 移除前缀：`支付宝-`、`微信支付-`、`财付通-`、`云闪付-`
- 正则去掉 `退款|退货|撤销`
- 匹配条件：**同商户 + 金额差 < 0.01** → 双方删除

## 支出分类规则（`_categorize`）

| 描述含 | 分类 | 子分类 |
|--------|------|--------|
| `饿了么` / `美团` / `叮咚` / `盒马` / `买菜` / `餐` / `食` | 食品酒水 | 早午晚餐 |
| `滴滴` / `高德` / `打车` / `出行` / `地铁` / `公交` | 行车交通 | 打车租车 |
| `淘宝` / `天猫` / `京东` / `拼多多` / `小红书` | 居家物业 | 日常用品 |
| `会员` / `vip` / `订阅` | 休闲娱乐 | 会员 |
