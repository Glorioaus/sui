## 输入格式

- CSV，**GBK 编码**，`pd.read_csv(file_path, encoding='gbk', skiprows=24)` —— **跳过前 24 行**。
- 列名 strip 清理。
- 账单周期从文件名 `\((\d{8})-(\d{8})\)` 提取。
- 列名：`交易时间`、`交易分类`、`交易对方`、`商品说明`、`收/支`、`金额`、`收/付款方式`、`交易订单号`、`交易状态`。

## 状态与收支常量（关键）

- `SKIP_STATUS = ["交易关闭"]` → 跳过
- `REFUND_STATUS = ["退款成功"]` → 走退款对冲
- `SUCCESS_STATUS = ["交易成功", "等待确认收货", "还款成功"]` → 等同交易成功（用于退款匹配的消费池）
- `BANK_KEYWORDS = ["银行", "信用卡"]`
- `WALLET_METHODS = ["余额", "余额宝", "花呗", "借呗"]`
- `收/支 = "不计收支"` → 跳过（避免与银行账单重复）

## 金额与收支规则

- 金额解析 `float(str.strip())`，为 0 跳过。
- 方向由 `收/支` 列：`支出`/`收入`，其他跳过。

## 退款对冲逻辑（`_handle_refund`，独立机制）

**两遍扫描**：

1. **第一遍**：把所有 `SUCCESS_STATUS` 且 `收/支="支出"` 的交易按 `(交易对方, 金额)` 存入 `expense_records` 字典。
2. **第二遍**：遇到 `退款成功`，用 `(交易对方, 金额)` 查字典：
   - **命中** → `del` 该消费（**对冲删除，不产记录**）
   - **未命中** → 产「其他收入/退款」收入记录，`description=退款 {对方}`

注意：这是**单文件内**对冲，与 merge.py 的跨文件对冲是两套机制。

## 亲友代付标记机制（关键，跨账单关联）

触发条件：`交易分类` 含 `亲友代付` **且** `收/支="支出"`。

生成**标记交易**：
- `category="__FAMILY_CARD__"`
- `transaction_type="__MARKER__"`
- `subcategory=使用者名`（取 `交易对方`，空则 `"亲友"`）
- `account=银行名 or "__ANY_BANK__"`
- `description="亲友代付 {使用者} {商品说明}"`

**银行名提取**：
- 含 `信用卡` → `(.+?)银行信用卡` + `信用卡`
- 含 `储蓄卡` → `(.+?)银行`

**注意**：支付宝的"交易对方"字段即为使用者姓名（如"张颖"）。

这些标记供 merge.py 的 `process_family_card` 处理，机制与微信亲属卡相同。

## 支付方式处理

| 支付方式 | 处理 |
|---------|------|
| 余额 / 余额宝 / 花呗 / 借呗 | 保留，记入"支付宝"账户 |
| 银行卡支出 | **跳过**（避免与银行账单重复） |
| 亲友代付 | 记录标记，供 merge 识别银行卡账单对应交易 |

## 描述拼接

`description = 商品说明（优先）或 交易分类`；交易对方非 `/` 则前置。
