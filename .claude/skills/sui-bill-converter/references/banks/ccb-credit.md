## 输入格式

- 账单周期从 `Statement Cycle\s+(\d{4})/(\d{2})/(\d{2})-(\d{4})/(\d{2})/(\d{2})` 取。
- **交易区判定**：从含 `[人民币账户]` 或 `RMB Account` 的行开始，到含 `结束` 且 `End` 的行结束（**该解析器独有的区间边界**）。
- **交易行正则**：`^(\d{4}-\d{2}-\d{2})\s+\d{4}-\d{2}-\d{2}\s+\d{4}\s+(.+?)\s+CNY\s+([-\d.]+)\s+CNY\s+([-\d.]+)$`
- 格式：`交易日 记账日 卡号 描述 CNY金额 CNY金额`
- **日期取交易日**（第一个）。

## 金额正负规则

- **正金额 = 消费支出**
- **负金额 = 退款或还款**

## 交易类型判定（`_process_refunds`）

| 条件 | 处理 |
|------|------|
| 负金额 且 含 `还款` | **跳过**（直接 skip，不保留 __REPAYMENT__ 标记） |
| 正金额 | 正常消费支出 |
| 负金额（其他） | `refunds` 退款对冲 |

**注意**：建行信用无"返现/红包转收入"分支。

## 商户提取与退款对冲（`_extract_merchant`）

前缀处理比其他信用卡复杂：

1. 移除嵌套前缀：`支付宝-支付宝-消费-`、`支付宝-`、`微信支付-`、`财付通-`、`云闪付-`（用 `description.split(prefix)[-1]`）
2. 移除地点+交易类型前缀：`re.search(r'(?:跨行消费|消费)\s+(.+)$', description)`
   - 例：`北京 跨行消费 京东商城平台商户` → `京东商城平台商户`
3. 正则去 `退款|退货|撤销`
4. 匹配条件：**同商户 + 金额差 < 0.01** → 双方删除
