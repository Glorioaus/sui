## 输入格式

- 头部前 10 行：用 `账户[：:]\s*(\d+)` 取账号，`(\d{8})-(\d{8})` 取起止日期。
- **交易行定位**：以 `^\d{8}\s`（8 位日期 + 空格）开头的行作为新交易起点；其后非日期开头的行追加到上一条（多行合并）。
- 字段顺序：`日期 [可选6位时间] 摘要 金额 余额 [对方/备注...]`
- 示例：`20251110 191619 转支 -15.00 21978.18 0051325048423775 1598069932 自动扣费还款-招行`

## 金额正负规则

**正金额 = 收入，负金额 = 支出**（取绝对值）。金额正则 `([+-]?\d+\.\d{2})`。

## 收入分类规则

`_apply_special_rules` 按 `text = (summary + description).lower()` 匹配，命中即返回：

| 描述含 | 分类 | 子分类 |
|--------|------|--------|
| `工资` / `代发` / `薪` | 职业收入 | 工资收入 |
| `公积金` / `gjj` | 职业收入 | 公积金转出 |
| `利息` / `结息` | 职业收入 | 利息收入 |
| `退款` / `退还` | 其他收入 | 退款 |
| `报销` | 其他收入 | 报销 |
| `张颖` / `小张` | 张颖转入 | 其他 |

## 支出分类规则

按 `text` 匹配，**注意有组合判定分支**：

| 描述含 | 分类 | 子分类 | 备注 |
|--------|------|--------|------|
| 账号 `39602057400003997`（字面量） | 金融保险 | 按揭还款 | **商贷贷款账号硬编码** |
| `贷款` / `按揭` / `房贷` | 金融保险 | 按揭还款 | |
| `住房公积金` 且（`代扣` 或 `还款`） | 金融保险 | 按揭还款 | 公积金冲还贷 |
| `公积金` / `gjj`（其他情况） | 居家物业 | 五险一金 | 普通公积金扣缴 |
| `微信` | 账单导入 | 微信账单导入 | |
| `支付宝` | 账单导入 | 支付宝账单导入 | |
| `短信费` | 交流通讯 | 手机费 | |
| `手续费` | 金融保险 | 银行手续 | |

未命中走通用 `match_category`（见 [classification.md](../classification.md)）。

## 硬编码业务常量

- 商贷贷款账号：`39602057400003997` —— 这是**业务语义识别**（不是隐私），LLM 无法凭描述语义推断，必须作为显式规则保留。
- 农行是储蓄卡源头，**不需要**跳过银行卡支付/退款对冲/亲属卡逻辑。
